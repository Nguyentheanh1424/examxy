# EDS v3.0 — Copywriting & Tone of Voice

## Principles

- **Professional but approachable**: Clear, concise, avoids technical jargon.
- **Positive framing**: "Save your exam" not "Don't forget to save your exam".
- **Benefit-focused**: Emphasize value, not just describe the feature.
- **Action-oriented**: Use verbs. "Create exam" not "Exam creation".

## Copy Patterns by Context

| Context             | Pattern                                     | Example                                                            |
| ------------------- | ------------------------------------------- | ------------------------------------------------------------------ |
| Validation error    | `[What's wrong] + [Specific guidance]`      | "Invalid email. Please enter a valid format (e.g. name@email.com)" |
| System error        | `[What happened] + [What user should do]`   | "Could not connect. Please try again in a moment."                 |
| Success             | `[Result] + [Next step if applicable]`      | "Submitted successfully! Scores will update shortly."              |
| Empty state         | `[Current state] + [Clear CTA]`             | "No exams yet. Create your first exam to get started."             |
| Loading             | `Progressive verb`                          | "Loading list…", "Analyzing exam…"                                 |
| Warning             | `[Impact] + [Action]`                       | "5 minutes left. Review your answers before submitting."           |
| Confirmation dialog | `[Consequence] + [Irreversibility if true]` | "Delete this exam? This action cannot be undone."                  |

## Words to Avoid

| Avoid                            | Use instead                                        |
| -------------------------------- | -------------------------------------------------- |
| "Error", "Bug", "Exception"      | "Something went wrong", "Could not complete"       |
| "Invalid" (alone)                | Be specific: "Invalid email address"               |
| "N/A", "Null", "None"            | "Not available", "No information yet"              |
| "Click" (mouse-only)             | "Tap" or "Select" (works for both touch and mouse) |
| "Submit" (generic)               | "Submit exam", "Confirm", "Save"                   |
| "Failed"                         | "Could not…", "Was not able to…"                   |
| "User" (referring to the person) | "You", "Student", "Teacher" (contextual)           |

## Button Label Conventions

| Action                       | Label                           |
| ---------------------------- | ------------------------------- |
| Create something new         | "Create [thing]" or "+ [Thing]" |
| Save changes                 | "Save changes"                  |
| Confirm a destructive action | "Delete [thing]" (be specific)  |
| Cancel / go back             | "Cancel" or "Go back"           |
| Submit an exam               | "Submit exam"                   |
| Retry after error            | "Try again"                     |
| Dismiss / close              | "Close" or "Dismiss"            |
| Navigate forward             | "Next" or "Continue"            |
| Navigate back                | "Back"                          |

## Empty State Title & Description Templates

```
No [items] yet
→ Create your first [item] to start [benefit].

No results for "[query]"
→ Try different keywords or clear your filters.

You don't have permission to view this
→ Contact your administrator if you think this is a mistake.

Could not load [content]
→ Something went wrong on our end. Try refreshing the page.
```
